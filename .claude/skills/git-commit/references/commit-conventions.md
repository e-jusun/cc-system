# 커밋 메시지 규칙

## 작업 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| `[Feature]` | 새로운 기능 추가 | `[Feature] 사용자 인증 기능 구현` |
| `[Fix]` | 버그 수정 | `[Fix] 로그인 시 토큰 만료 오류 해결` |
| `[Refactor]` | 코드 리팩토링 | `[Refactor] API 호출 로직 모듈화` |
| `[Docs]` | 문서 업데이트 | `[Docs] README 설치 가이드 추가` |
| `[Test]` | 테스트 추가/수정 | `[Test] 결제 모듈 유닛 테스트 추가` |
| `[Chore]` | 빌드/설정 변경 | `[Chore] ESLint 규칙 업데이트` |
| `[Style]` | 코드 스타일 변경 | `[Style] 코드 포맷팅 정리` |
| `[Perf]` | 성능 개선 | `[Perf] 이미지 로딩 최적화` |

## 제목 작성 규칙

1. **50자 이내**로 간결하게
2. **명령형**으로 작성 ("추가", "수정", "삭제")
3. 마침표 없이 끝내기
4. **무엇을** 했는지 명확히

**좋은 예:**
- `[Feature] OAuth 2.0 소셜 로그인 추가`
- `[Fix] 장바구니 수량 동기화 오류 수정`

**나쁜 예:**
- `[Feature] 버그 수정함.` (모호함, 마침표)
- `여러 가지 수정` (유형 없음, 모호함)

## 상세 내용 (선택)

제목만으로 부족할 때 본문 추가:

```
[Feature] 결제 시스템 리뉴얼

상세 변경사항:
- payments/checkout.ts: 새 결제 플로우 구현
- payments/validation.ts: 카드 유효성 검증 추가
- types/payment.d.ts: 결제 관련 타입 정의

관련 이슈: #123
```

## 본문 작성 팁

1. **왜** 이 변경이 필요했는지 설명
2. 주요 변경 파일과 내용 나열
3. 관련 이슈/티켓 번호 포함
4. 빈 줄로 제목과 본문 구분
