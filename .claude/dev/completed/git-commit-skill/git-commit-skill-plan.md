# git-commit-skill 구현 계획

**Last Updated**: 2026-01-01 12:05

## 목표

`.claude/commands/dev:git-commit.md` 슬래시 커맨드를 범용화하여 `~/.claude/skills/git-commit/` skill로 변환. 프로젝트 특화 로직 제거, 범용 패턴 적용.

## 구현 단계

1. skill-creator로 기본 구조 생성
2. SKILL.md 작성 (핵심 워크플로우)
3. scripts/find_test_files.py 생성 (범용 테스트 파일 탐색)
4. references/commit-conventions.md 생성 (커밋 메시지 규칙)
5. 패키징 및 테스트

## 기술적 접근

### 제거할 것 (프로젝트 특화)
- TSV 검증 로직 (`script/validate_tsv.py`, game_type 판단)
- 하드코딩된 경로 (`script/test/`, `script/test/test_output/`)
- 하드코딩된 submodule (`pd_prototyping`)
- AC3 서명

### 범용화할 것
| 현재 | 범용화 |
|------|--------|
| `script/test/` 경로 | 범용 패턴 (`test/`, `tests/`, `**/test_*`) |
| `pd_prototyping` 하드코딩 | `git submodule status` 자동 감지 |
| "Generated by AC3" | "Generated by Merak" |
| TypeScript 체크 | tsconfig.json 있으면 자동 실행 |

## 예상 산출물

### 생성될 파일
- `~/.claude/skills/git-commit/SKILL.md`: 핵심 워크플로우
- `~/.claude/skills/git-commit/scripts/find_test_files.py`: 범용 테스트 파일 탐색
- `~/.claude/skills/git-commit/references/commit-conventions.md`: 커밋 메시지 규칙

## 고려사항

- 설정 파일 없이 범용화 (하드코딩된 값 사용)
- Submodule 자동 감지로 유연성 확보
- TypeScript 프로젝트 자동 감지

## 검증 방법

- 스크립트 실행 테스트
- 패키징 성공 확인
